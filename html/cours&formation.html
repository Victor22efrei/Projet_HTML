Cours HTML, CSS, JavScript:

HTML — LA STRUCTURE

Le HTML sert à construire le contenu du site.
C’est la base : sans HTML, rien ne s’affiche.

Une page est toujours composée de deux parties : le head : informations invisibles (titre, encodage…) le body : tout ce qui est visible à l’écran

Le HTML fonctionne avec des balises qui permettent d’organiser le contenu : titres (importance du contenu) paragraphes (texte) images liens listes tableaux formulaires (interaction utilisateur)

Il existe aussi des balises modernes pour structurer correctement un site : header, nav, section, article, footer
Une règle très importante : toutes les balises doivent être correctement ouvertes et fermées pas de chevauchement les attributs donnent des infos supplémentaires (id, class…)
Enfin, une page HTML est représentée comme un arbre (DOM), ce qui permet de la manipuler avec JavaScript.

À retenir : HTML = structure + organisation du contenu

CSS — LE DESIGN

Le CSS sert à styliser le site, c’est-à-dire à rendre le site beau et lisible. Il permet de : changer les couleurs modifier les tailles de texte choisir les polices ajouter des bordures organiser la mise en page Le CSS fonctionne avec des règles composées de : un sélecteur (ce que tu veux modifier) des propriétés (ce que tu changes) Il existe plusieurs types de sélecteurs : balise (tous les éléments d’un type) classe (plus précis) id (unique) Un concept très important est la priorité des règles : la règle la plus spécifique est appliquée si deux règles sont égales, la dernière écrite gagne Le CSS utilise aussi le modèle de boîte : margin : espace extérieur padding : espace intérieur border : bordure Pour la mise en page moderne, on utilise flexbox, qui permet d’aligner facilement les éléments. Le responsive design est essentiel : le site doit s’adapter aux téléphones, tablettes et ordinateurs. Pour ça on utilise : des tailles relatives (comme les pourcentages) les media queries (règles selon la taille de l’écran) À retenir : CSS = apparence + mise en page + adaptation écran

JAVASCRIPT — L’INTERACTION

Le JavaScript sert à rendre le site dynamique. C’est lui qui permet : de réagir aux clics de modifier la page en temps réel de créer des animations de gérer les formulaires Le code est exécuté directement par le navigateur. Les bases du JavaScript : variables pour stocker des données conditions pour prendre des décisions boucles pour répéter des actions fonctions pour organiser le code Le concept le plus important est celui des événements : JavaScript attend une action de l’utilisateur (clic, saisie, mouvement…) et réagit. Autre notion clé : le DOM JavaScript peut accéder aux éléments HTML et les modifier : changer du texte ajouter ou supprimer des éléments modifier le style Il existe aussi des fonctions utiles : afficher des messages rediriger vers une autre page exécuter du code après un délai Enfin, JavaScript peut gérer des actions asynchrones : certaines actions se font en parallèle sans bloquer le reste du programme (ex : attendre une réponse serveur).

Cours Initiation à Java:

  1. Principe de la POO

    La programmation orientée objet permet d’organiser un programme autour d’objets. Un objet regroupe : des données (attributs) des comportements (méthodes) Elle améliore : la lisibilité la modularité la maintenabilité des programmes Les quatre concepts fondamentaux sont : encapsulation abstraction héritage polymorphisme

  2. Classe et objet

    Une classe est une abstraction représentant un ensemble d’objets ayant les mêmes caractéristiques. Elle contient : des attributs (propriétés) des méthodes (actions) Un objet est une instance d’une classe, c’est-à-dire une réalisation concrète de cette classe

  3. Structure d’un programme Java

    Chaque classe est définie dans un fichier distinct Le programme principal est contenu dans une classe comportant la méthode main Une classe comporte généralement : des attributs un ou plusieurs constructeurs des méthodes

  4. Les attributs

    Les attributs sont des variables décrivant l’état d’un objet. Si un attribut n’est pas initialisé, Java lui attribue une valeur par défaut : int : 0 double : 0.0 boolean : false objet : null

  5. Les constructeurs

    Un constructeur est une méthode particulière permettant de créer et initialiser un objet. Caractéristiques : porte le même nom que la classe ne possède pas de type de retour est appelé automatiquement lors de l’instanciation Types de constructeurs : constructeur par défaut (sans paramètres) constructeur avec paramètres (initialisation directe) Point important : Si un constructeur est défini explicitement, le constructeur par défaut n’est plus généré automatiquement. Il doit alors être redéfini si nécessaire

  6. Instanciation et gestion mémoire

    L’instanciation permet de créer un objet. Lorsqu’on affecte un objet à un autre : les deux variables pointent vers la même zone mémoire Toute modification via l’un affecte l’autre. Solution : Utiliser un constructeur de copie pour créer une duplication réelle de l’objet

  7. Encapsulation

    L’encapsulation consiste à : regrouper données et méthodes dans une classe protéger les données internes Niveaux de visibilité : public : accessible partout private : accessible uniquement dans la classe protected : accessible dans la classe et ses sous-classes Bonne pratique : attributs en private méthodes publiques pour y accéder

  8. Getters et Setters

    Ils permettent d’accéder et de modifier les attributs privés : getter : lecture d’un attribut setter : modification d’un attribut Ils assurent le respect de l’encapsulation

  9. Affichage d’un objet

    Par défaut, Java affiche une référence mémoire. Pour afficher les informations d’un objet, on redéfinit la méthode : toString() Cette méthode retourne une représentation textuelle de l’objet

  10. Annotation @Override

    L’annotation @Override indique qu’une méthode redéfinit une méthode héritée. Elle permet : d’éviter les erreurs de signature de vérifier la cohérence du code

  11. Comparaison d’objets

    L’opérateur == compare les références mémoire. Pour comparer le contenu de deux objets, on utilise la méthode : equals() Cette méthode doit être redéfinie pour comparer les attributs

  12. Héritage

    L’héritage permet de créer une classe à partir d’une autre. classe mère (superclasse) classe fille (sous-classe) Avantages : réutilisation du code réduction de la duplication En Java : une seule classe mère (pas d’héritage multiple) Redéfinition (Overriding) Une classe fille peut redéfinir une méthode héritée : même nom mêmes paramètres Le mot-clé super permet d’appeler la version de la classe mère

  13. Polymorphisme

    Le polymorphisme permet de manipuler différents objets à travers une même interface. UpCast conversion d’un objet fils vers un type parent implicite DownCast conversion inverse explicite et plus risquée Utilité : manipuler différents objets dans une structure commune (ex : tableau)

  14. Classe Scanner

    La classe Scanner permet de lire des données : au clavier dans une chaîne dans un fichier Méthodes principales : nextInt() nextLine() nextDouble() Des méthodes de vérification existent (hasNext…) pour sécuriser la lecture Conclusion La programmation orientée objet repose sur une structuration claire : Classe : modèle Objet : instance Attribut : état Méthode : comportement Les concepts essentiels : Encapsulation : protection des données Héritage : réutilisation Polymorphisme : flexibilité Cette approche permet de développer des programmes plus robustes, évolutifs et maintenables.

Cours Bases de Données :

La conception d’une base de données repose sur une démarche structurée appelée méthode MERISE, qui permet de passer progressivement d’un besoin métier à une base exploitable techniquement. Dans un premier temps, on réalise une analyse afin de comprendre les besoins, les données à stocker et les contraintes. À partir de cette analyse, on construit le Modèle Conceptuel de Données (MCD), qui constitue une représentation simple et visuelle du système. Ce modèle est ensuite transformé en Modèle Logique de Données (MLD), plus formel, puis enfin en modèle physique implémenté dans un système de gestion de base de données à l’aide du langage SQL.

Le MCD est une étape essentielle car il permet de représenter les données de manière compréhensible, même pour des personnes non techniques. Il repose sur le modèle entité-association. Une entité correspond à un objet du monde réel que l’on souhaite modéliser, comme un client, un produit ou un film. Chaque entité possède des attributs, c’est-à-dire des propriétés qui la caractérisent, par exemple un nom, une date ou un prix. Certaines propriétés peuvent être simples, d’autres composées ou encore contenir plusieurs valeurs. Chaque entité doit également posséder un identifiant, qui permet de distinguer de manière unique chaque instance. Cet identifiant doit être stable, unique et de préférence simple pour garantir de bonnes performances.

Les entités sont reliées entre elles par des associations, qui représentent des liens logiques entre les objets. Par exemple, un client peut passer une commande ou un acteur peut jouer dans un film. Ces associations peuvent relier deux entités ou plus, et peuvent même être réflexives, c’est-à-dire relier une entité à elle-même. Pour chaque association, on définit des cardinalités, qui indiquent combien de fois une entité peut intervenir dans la relation. Ces cardinalités permettent de traduire les règles métier, par exemple le fait qu’un client peut passer plusieurs commandes, mais qu’une commande est liée à un seul client.

Le passage du MCD au MLD consiste à transformer cette représentation conceptuelle en une structure exploitable sous forme de tables. Dans ce modèle, chaque entité devient une table, chaque attribut devient une colonne et l’identifiant devient la clé primaire. Les associations entre entités sont traduites par des clés étrangères ou, dans certains cas comme les relations plusieurs-à-plusieurs, par la création d’une table supplémentaire. Le modèle relationnel ainsi obtenu repose sur des règles strictes garantissant la cohérence et l’intégrité des données, notamment grâce aux contraintes et aux liens entre tables.

Enfin, une fois la base de données structurée, il devient possible de manipuler les données grâce au SQL, qui est le langage standard utilisé pour interroger les bases. Ce langage permet notamment de sélectionner des données, de filtrer des résultats, de combiner plusieurs tables ou encore de calculer des statistiques. Ces opérations reposent sur les principes de l’algèbre relationnelle, qui définit les différentes manipulations possibles sur les données, comme la sélection, la projection ou les jointures.

En résumé, la conception d’une base de données consiste à partir d’un besoin réel, à structurer les données de manière logique et cohérente, puis à les rendre exploitables techniquement. Le MCD permet de comprendre le système, le MLD permet de le structurer et le SQL permet de l’utiliser efficacement.
